産業組織：価格設定の経済学

Kei Kawai

専門：産業組織論（Industrial Organization）
企業の価格設定戦略、オークション・入札、産業の競争構造を実証的に分析。 普段は英語での授業が主であり、本講義は日本語で実施。

講義概要

企業はなぜ経済学者を雇うのか？最も重要な理由は適切な価格設定を任せるためだ。 コスト＋20%でも、感覚的な500%でもなく、利益を最大化する価格を計算するには需要関数の推定が必要になる。 本講義では、ZipRecruiter.com（求人マッチングプラットフォーム）での価格実験を事例として、 需要関数の推定から最適価格の導出、さらに個別価格設定（価格差別）まで、産業組織論の実証的アプローチを学ぶ。

    産業組織論（I.O.）の守備範囲

    産業組織論（Industrial Organization, I.O.）は、企業の意思決定と消費者行動を分析する経済学の一分野。 企業サイドでは：価格設定・広告量の決定・市場への参入と退出・設備投資の判断。 消費者サイドでは：価格や広告への反応（需要の価格弾力性など）。 競争政策・独占禁止法の立案・執行と密接に関係しており、経済学のスキルが実際の政策や企業戦略に直結する分野。 マーケティング（個別企業視点）・経営学（企業行動の戦略的分析）と重なる部分が多い。

    I.O.は「現実の市場でどう競争が行われているか」を問う実証科学。完全競争モデルでは解けない現実の問いに答えるために、計量経済学・ゲーム理論・実験的手法を組み合わせる。
  • 2
    企業はなぜ経済学者を雇うか

    企業が経済学者を採用する最大の理由は「適切な価格設定」のため。 価格は企業にとって最もコントロールしやすい変数であり、かつ最も利益インパクトが大きい意思決定のひとつ。 製品開発・工場建設・広告キャンペーンはコストと時間がかかるが、価格変更は即座に実行できる。 しかし「いくらに設定すべきか」を正確に判断するには、消費者がどう反応するかを数値で把握する必要がある——これが需要関数の推定。

    利益最大化の基本式：π = (P − c) × q(P)
    P：価格、c：限界費用、q(P)：需要量（価格の関数）
    最適価格はこれを最大化するP*。需要関数 q(P) を知らなければ計算できない。

    価格変更の利益インパクトは、製品開発や広告投資に比べて費用対効果が圧倒的に高い（数割のオーダーになることも）。
  • 3
    ZipRecruiter.com の事例：オンライン求人マッチング

    ZipRecruiter.com はオンライン求人マッチングプラットフォーム。 求職者（無料で利用）と雇用者（手数料を払って求人掲載）を結びつける。 延べ300万社以上・1億人以上が利用する大規模プラットフォーム。 雇用者向けの月額サブスクリプション価格をいくらに設定するか——これが分析の課題。 同様の問いは、あらゆるSaaS・プラットフォーム企業が直面している。

    プラットフォームビジネスでは「価格設定が需要の多面性（雇用者と求職者）に影響する」点が特殊。雇用者価格を上げすぎると求人が減り、求職者の魅力も下がる負のスパイラルが生じうる。
  • 4
    価格実験：ランダム化実験による需要推定

    需要関数を推定する最もクリーンな方法は「ランダム化実験（A/Bテスト）」：異なる価格をランダムに提示し、購入率の差を観察する。 ZipRecruiterの実験設計：訪問者をランダムに3グループに分け、月19ドル・99ドル・400ドルをそれぞれ提示。

    実験結果（概数）：
    19ドル提示 → 約35%が購入（購入確率 q(19)≈0.35）
    99ドル提示 → 約25%が購入（q(99)≈0.25）
    400ドル提示 → 約15%が購入（q(400)≈0.15）

    期待売上：19×0.35 = $6.65、99×0.25 = $24.75、400×0.15 = $60.00
    → 期待売上だけ見ると400ドルが最も高い！しかし限界費用や顧客ライフタイムバリューも考慮が必要。
    ランダム化実験の威力：価格が違う理由が「純粋にランダム」なので、需要の差が価格だけによるものだと因果的に言える。観察データで「高い価格の商品が売れている」ことを見てもそれは品質の差かもしれず、因果推論が難しい。
  • 5
    需要関数の推定と最適価格の導出

    実験で得た3点（価格・購入確率のペア）から需要関数 q(P) をフィットし、利益を最大化する価格 P* を計算する。 コストがゼロなら期待売上 P × q(P) を最大化、コスト c がある場合は期待利益 (P − c) × q(P) を最大化する。

    一次の最適条件（FOC）：d/dP [(P − c) × q(P)] = 0
    q(P*) + (P* − c) × q'(P*) = 0
    → P* − c = − q(P*) / q'(P*) = P* / ε（εは需要の価格弾力性）

    ZipRecruiterの結論：実験1の分析では99ドルは安すぎた。最終的に月249ドルに引き上げて利益アップを実現。単純なコスト積み上げではなく、需要の価格感応度（弾力性）から逆算して価格を設定することが鍵。
  • 6
    個別価格設定：企業特性に基づく価格差別

    全顧客に同じ最適価格を設定するより、顧客ごとに異なる価格を設定する方が利益が高い——これが価格差別（Price Discrimination）の論理。 ZipRecruiterでは企業の規模・業種・従業員数などの特性（x）に基づいて個別価格を設定する。

    個別価格設定の計量経済学：
    各企業のG(x)（需要曲線の傾き：価格感応度）とF(x)（需要曲線の切片：購買意欲）を特性 x から推定する。
    → 同じ価格でも大企業と小企業では価格感応度が異なる（大企業は高価格でも購入する）。

    価格差別が利益を増やす理由：消費者余剰（支払意思額と実際価格の差）を企業が吸収できるため。ただし、細分化するほどデータ不足が問題になる。
    経済厚生の観点：価格差別は企業利益を増やす一方、公平性の問題を生じさせる。同じサービスに異なる価格を払うのは「不公平」か？これは社会的・政治的な問いでもある。
  • 7
    価格設定の多様な形態

    単一価格設定は最もシンプルだが、以下のような複雑な価格設定が現実には広く用いられる。

    段階的価格設定（ブロック料金）：一定量まで無料・超過分は有料（水道料金モデル）。また「フリーミアム」（基本無料、プレミアム機能は有料）もこの変形。

    ダイナミックプライシング：需要に応じてリアルタイムで価格を変動させる。航空・ホテル・Uberなどが典型。需要が高い時間帯に高く、低い時間帯に安くする。

    サブスクリプション設計：繰り返し購入する顧客に対して、1回払いよりも割安な月額・年額を設定することで顧客固定を図る。解約率（チャーン）の最小化とLTV（顧客生涯価値）の最大化が目標。

    バンドル価格：複数商品を組み合わせて割安に販売。顧客の評価が商品間で相関していれば有利（Officeスイートなど）。
  • 8
    実証産業組織論の研究フロンティア

    実証I.O.（Empirical Industrial Organization）は、価格設定にとどまらず、企業行動のあらゆる側面を計量経済学・実験・ゲーム理論を組み合わせて分析する。

    入札・調達市場の研究：公共調達のオークション設計。入札者が談合しているか、落札価格は競争的かを分析。Kawai教授の主要研究領域。

    競争政策への応用：合併・買収が競争に与える影響をシミュレート（競争当局の審査に使われる）。独占力の測定（ラーナー指数）。

    デジタルプラットフォームの規制：GAFA等のプラットフォームの価格設定・市場支配力の評価が現代I.O.の最前線。「ゼロ円のビジネスモデル」でも消費者に広告費という隠れたコストがかかる。
    I.O.研究者の就職先：学術（大学・研究機関）だけでなく、大企業の経済分析チーム（アマゾン・グーグル・マイクロソフト）、コンサルティング（Charles River Associates）、規制当局（公正取引委員会・FTC）など多様。

各キーワードの解説（クリックで展開）

産業組織論（I.O.）
企業の意思決定と消費者行動を分析する経済学の一分野。第7回（規制・交通）に続き今回は企業の価格設定に焦点。競争政策・独占禁止法の立案・執行と密接に関係し、大企業の経済分析チームへの就職にも直結する。
需要関数
価格と需要量の関係 q(P)。企業が最適価格を計算するには需要関数を知ることが不可欠。これを推定するためにランダム化実験（A/Bテスト）を使うのが今回の核心的手法。「需要関数なしに最適価格は計算できない」。
価格弾力性
価格が1%上がると需要が何%下がるかを表す指標（ε）。最適価格の条件式は P*−c = P*/ε——弾力性が高い（価格に敏感）ほど最適価格は限界費用に近づく。需要関数から数値として推定できる。
価格実験（A/Bテスト）
異なる価格をランダムに提示し購入率の差を観察することで需要関数を因果的に推定する手法。ZipRecruiterは19・99・400ドルの3点でランダム提示を実施。「価格設定に経済学者を雇う最大の理由」。
ランダム化実験
価格の違いが「純粋にランダム」なので需要の差は価格のみによる因果効果と言える。観察データでは「高価格品が売れている＝品質が高いから」という逆因果が入り込むため因果推論が難しい。開発経済学のRCTと同じ発想。
ZipRecruiter
オンライン求人マッチングプラットフォーム（雇用者有料・求職者無料）。雇用者向け月額サブスクリプション価格をいくらに設定するかという実問題でランダム化価格実験を実施。結論：99ドルは安すぎた→249ドルへ引き上げ。
利益最大化
π = (P−c)×q(P) を最大化する P* を求めること。一次条件：q(P*) + (P*−c)q'(P*) = 0。コスト積み上げでも感覚的な値付けでもなく、需要の価格感応度（弾力性）から逆算するのが経済学的アプローチ。
価格差別
全顧客に同一価格ではなく顧客ごとに異なる価格を設定する戦略。大企業は高価格でも購入する（価格感応度が低い）ため、均一価格より利益が高い。消費者余剰（支払意思額−実際価格）を企業が吸収できる。公平性の問題も。
ダイナミックプライシング
需要に応じてリアルタイムで価格を変動させる仕組み。航空・ホテル・Uberが典型。第7回のピークロード・プライシングの現代的応用。需要が高い時間帯に高く、低い時間帯に安くすることで収益と需要管理を両立。
サブスクリプション
繰り返し購入する顧客向けの月額・年額定額サービス。解約率（チャーン）最小化とLTV（顧客生涯価値）最大化が設計の鍵。ZipRecruiterはこのビジネスモデル。SaaS・ストリーミングサービス等で広く採用。
バンドル価格
複数商品を組み合わせて割安に販売する戦略。顧客の評価が商品間で相関していれば単品販売より利益増。Microsoft Office スイートが典型例。消費者余剰の吸収効率を高める価格設定の変形。
消費者余剰
「支払意思額（評価額）− 実際の価格」の合計。価格差別はこれを企業が吸収するために行われる。価格差別が完全（第一級価格差別）なら消費者余剰はゼロになる。競争政策では消費者余剰の保護が重要目標。
競争政策
合併・買収が競争に与える影響の評価、独占力の測定（ラーナー指数）、GAFA等のプラットフォーム規制。I.O.研究が規制当局（公正取引委員会・FTC）の意思決定を支え、Kawai教授の主要研究領域でもある。

Q7価格最適化・実験（誤文選択）
Zip recruiterの例：実験期間中は$19〜$399の価格がランダムに割り振られた。実験前は$99/mo。次の中から間違っているものを選んでください。
正解（誤りの選択肢）(c)：ランダムな価格をつけていた実験期間中の一社当たりの期待利潤は実験前の期待利潤よりも低かった。
$99は最適価格より低かった可能性が高い（最適は$175程度）。$19〜$399のランダム価格の期待値は$99より高い→実験期間中の方が実験前より利潤が高かった可能性がある。(a)は正しい（最適価格$175より平均的に劣る）、(b)は正しい（最適価格設定後は実験期間中より利潤大）。

Q11需要関数の定義
需要関数とは
正解 (c)：価格と販売量の関係をあらわしたもの
需要関数 Qd = D(P) は財の価格と消費者が購入したい数量（需要量）の関係。通常、価格が上がると需要量は減る（右下がりの需要曲線）。「在庫と販売量」は供給側の概念、「効用と販売量」は需要関数の背景概念で直接の関係ではない。
Q12価格最適化・計算
コンバージョンと価格の関係が conversion (%) = 0.35 − 0.001 × price で与えられたとして、売上が一番大きくなる price を選択しなさい。
正解 (b)：175
売上 Revenue(p) = p × (0.35 − 0.001p)。dRevenue/dp = 0.35 − 0.002p = 0 → p* = 175。確認：Revenue(175) = 175 × 0.175 = 30.6。(100の場合25、200の場合30、300の場合15で175が最大。)
